finding the devil with open refine

The devil has been a popular theme of modern music since Robert Johnson sold his soul to him in the 1930s. In the late 1980s and early 1990s, a moral panic concerning Satanism and alleged Satanism-related crimes swept the United States. In this poorly-methodologized example purely for entertainment purposes which in no way bears resemblance to actual rigorous research, I will get a data set into shape using Open Refine to see whether the spike in Satanic Panic correlates with a spike in music about the devil.

get the stuff

the tools

Using OpenRefine is a good way to find commonalities in a large data set by using facets and filters to act upon different parts of it. This is much faster than loading it into Excel or another software program, and you can load files directly from the web. Get OpenRefine, and view some great screencast tutorials.

the data

We have to find a data set that will help me do my analysis. The Million Song Dataset contains a million songs from 1930 to 2011. You can read more about the methodology of gathering these songs and their metadata at the data set's documentation page.

the data?

One key bit of information I found when looking through the MSD documentation: they only have year metadata for about half the tracks, and they make this available in an additional data file that’s just a text file. This is bad because it’s fewer data points to work with for my analysis, but good because I don’t have to learn a new data format and processing method to be able to use the data.

Make sure to check the options for ingest thoroughly -- for this data set, we don't want to split columns that contain a " (that will break everything!) and we have no headers.

de-duplicate records

This part is tricky. The “Duplicates” facet will let you filter out duplicates, but won’t let you leave one instance of a record in. I definitely want one version of a track in my data set, but not two or three. To remove subsequent instances of duplicates, do this:

Sort on the column that has duplicates. Reorder rows permanently (the blue SORT link which appears in the header) (this will take some time) Edit cells — BLANK down Face — customized facets — facet by blank All — edit rows — remove all matching rows — this will get rid of them so they aren’t even in the set anymore. You can undo anything at any time.
